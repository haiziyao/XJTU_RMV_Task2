转化为灰度图<br/> - 转化为 HSV 图片<br/>- 应用各种滤波操作<br/> - 应用均值滤波<br/> - 应用高斯滤波<br/>- 特征提取<br/> - 提取红色颜色区域<br/> - HSV 方法<br/> - 寻找图像中红色的外轮廓<br/> - 寻找图像中红色的 bounding box<br/> - 计算轮廓的面积<br/> - 提取高亮颜色区域并进行图形学处理<br/> - 灰度化<br/> - 二值化<br/> - 膨胀<br/> - 腐蚀<br/> - 对处理后的图像进行漫水处理<br/>- 图像绘制<br/> - 绘制任意圆形方形和文字<br/> - 绘制红色的外轮廓<br/> - 绘制红色的 bounding box<br/>- 对图像进行处理<br/> - 图像旋转 35 度。<br/> - 图像裁剪为原图的左上角 1/4。<br/><br/>:::tip 提示<br/>程序应生成处理后的图像文件并保存。代码需结构清晰，添加**适当的注释**以提高可读性。<br/>:::<br/><br/>#### 项目构建与运行<br/><br/>```bash<br/>cd opencv_project/build<br/>cmake ..<br/>make -j<br/>./OpenCV_Project<br/>```<br/><br/>运行后，程序应生成处理后的图像文件，并在终端输出各轮廓的面积信息。<br/><br/>#### README 报告<br/><br/>本任务依然要求组员提交 README 报告，包括自己完成任务的思路，从何处寻找的资料，如向 GPT 寻求帮助，可以提供与 GPT 的聊天记录。<br/><br/>如有报错等内容，也请在 README 中说明。<br/><br/>本任务主要重点在于程序的编写，组员无需过于关注 README 的格式问题，但是请提交符合 Markdown 语法的易读的 README，这很重要。<br/><br/>## 提交内容<br/><br/>整体的 opencv_project 项目需提交至 Github，其中应包括，符合 `项目结构` 的内容，即至少包括以下内容：<br/><br/>1. **源代码**<br/>2. **资源文件**：<br/> -
